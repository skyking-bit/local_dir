🔌 CXL & PCIe Protocol Questions

Q1
What is CXL (Compute Express Link)? Explain device types.
HardKeysight▼

CXL is an open industry-standard, cache-coherent interconnect protocol built on top of PCIe's physical layer. It solves how CPUs and accelerators (GPUs, FPGAs, AI chips) can efficiently share memory with low latency and coherency.

Analogy: PCIe is a highway for data transport. CXL adds traffic rules (coherency protocols) so devices share memory like teammates, not strangers.

3 Sub-protocols:

  • CXL.io — Based on PCIe. Device discovery, config, interrupts, DMA. Required for ALL types.
  • CXL.cache — Device caches host CPU memory. Ensures cache coherency.
  • CXL.mem — CPU accesses device-attached memory (HBM on FPGA) with byte-granularity.
TypeProtocolsUse CaseLocal Memory
Type 1CXL.io + CXL.cacheSmart NICs, Accelerators❌ No
Type 2CXL.io + CXL.cache + CXL.memGPUs, FPGAs, ASICs✅ Yes (DDR/HBM)
Type 3CXL.io + CXL.memMemory Expanders✅ DRAM/NVRAM
Interview Trick: Type 1=cache only, Type 2=cache+mem, Type 3=mem only. CXL.io is ALWAYS required! PCIe latency ~500ns vs CXL ~20–40ns.
Q2
What is InitFC1/FC2 and what happens when credits = ZERO?
HardKeysight▼

PCIe Flow Control prevents a transmitter from sending more data than the receiver can buffer.

  • InitFC1: Each side advertises available buffer space for Posted (P), Non-Posted (NP), Completions (Cpl).
  • InitFC2: Response acknowledging InitFC1. Both sides echo credit values. DL_Up state reached.
  • UpdateFC: Sent periodically to return credits as buffers are consumed.

6 Credit Types: PHC, PDC (Posted), NPHC, NPDC (Non-Posted), CpHC, CpDC (Completion)

⚠️ EXAM TRAP:
Credit = 0 for Completions → INFINITE credits (endpoints MUST advertise 0)
Credit = 0 for Posted/Non-Posted → STALL! Transmitter cannot send.

Q4
What is Scrambling in PCIe and why is it needed?
MediumSiemens▼

Scrambling XORs the data stream with a pseudo-random binary sequence (PRBS) before transmission.

  • EMI Reduction: Spreads energy across frequencies, reducing EMI peaks
  • DC Balance: For 128b/130b (Gen3+), replaces the role 8b/10b played in Gen1/2
  • Clock Recovery: Regular transitions help CDR circuits at the receiver
  • Better BER: Spreading spectrum improves signal integrity
Implementation: PRBS-7 or PRBS-23 sequences via LFSR. TX and RX reset LFSRs simultaneously using sync markers.

Q6
What is PCIe Bifurcation?
MediumIBM▼

PCIe Bifurcation splits a single physical PCIe slot into multiple independent links.

  • x16 → 2× x8 (most common for dual NVMe)
  • x16 → 4× x4 (quad NVMe)
  • x16 → 1× x8 + 2× x4 (mixed)
Use Case: M.2 NVMe SSDs in a PCIe x16 slot. Configured in BIOS/UEFI or via PCIe switches.
⏱ Timing Analysis Questions
Q7
Explain Setup and Hold Time Violations and how to resolve them.
HardSiemens▼
Metastability: Violating setup/hold causes output to be neither 0 nor 1 — it can oscillate unpredictably!

Setup Time (Ts): Data must be stable BEFORE clock edge by at least Ts.

Tclk > Tcq + Tcomb + Tsetup + Tskew
Slack = Tclk − (Tcq + Tcomb + Tsetup + Tskew)

Hold Time (Th): Data must remain stable AFTER clock edge for at least Th.

Tcq + Tcomb > Thold + Tskew
Slack = Tcq + Tcomb − Thold − Tskew
Fix Setup Violation:
• Reduce combinational logic depth
• Increase clock period
• Insert pipeline registers
• Use faster standard cells
• Reduce clock skew
• Use retiming
Fix Hold Violation:
• Add buffer/delay cells on data path
• Use larger cells with more delay
• Increase clock skew intentionally
• Cannot fix by changing frequency!
• Hold is clock-period-independent
Q8
Fanout Reduction in Digital Circuits — why and how?
MediumIBM▼

High fanout means one signal drives many loads → increased capacitance → slower transitions → timing violations.

  • Use buffer trees — insert buffers to split the load
  • Clock tree synthesis (CTS) for clock signals
  • Use higher drive strength cells for critical nets
  • Replicate logic — duplicate the driving gate for different load groups
  • Set max_fanout constraints in synthesis tool
Rule of Thumb: Fanout > 16–20 usually needs buffering. Clock nets target fanout of 4–8 per buffer stage in CTS.

📦 FIFO Design & Clock Domain Crossing
Q11
FIFO Depth Calculation — Formula & Examples (Synopsys Interview)
HardSynopsys▼
FIFO Depth = Write_words − Read_words during burst window

Example 1: Write=200MHz, Read=100MHz, burst=100 cycles

Words written = 100 | Words read = 50 | Depth = 50 words

Example 2 (Tricky — Synopsys): Write=80 bits/100 clk, Read=8 bits/10 clk

Rates are EQUAL (0.8 bits/clk)! But worst case: all 80 bits arrive in first clock → FIFO = 80 bits = 10 bytes

Example 3: Wr=400MHz, Rd=100MHz, 20 packets × 8 bytes

Write burst = 160 bytes | Read in 50ns = 5 bytes | Depth = 155 bytes ≈ 160
Always: Round up to next power of 2. Add 2–3 extra entries for async FIFO gray-code sync latency.

Sync FIFO — Core Logic
module sync_fifo #(parameter DEPTH=16, WIDTH=8) ( input clk, rst, wr_en, rd_en, input [WIDTH-1:0] din, output reg [WIDTH-1:0] dout, output full, empty ); reg [WIDTH-1:0] mem [0:DEPTH-1]; reg [$clog2(DEPTH):0] wr_ptr, rd_ptr; assign full = (wr_ptr - rd_ptr) == DEPTH; assign empty = (wr_ptr == rd_ptr); always @(posedge clk or posedge rst) begin if (rst) begin wr_ptr <= 0; rd_ptr <= 0; end else begin if (wr_en && !full) begin mem[wr_ptr[3:0]] <= din; wr_ptr <= wr_ptr+1; end if (rd_en && !empty) begin dout <= mem[rd_ptr[3:0]]; rd_ptr <= rd_ptr+1; end end end endmodule
Q13
Clock Domain Crossing (CDC) — Types & Techniques
HardSiemens▼

CDC occurs when a signal from clk_A is sampled in clk_B with no fixed timing relationship → Metastability risk!

CDC TypeSolutionNotes
Single-bit control2-FF SynchronizerMost common. 3-FF for >1GHz
Multi-bit data busFIFO + Gray pointers / HandshakeNEVER use 2-FF on bus directly!
Short pulseToggle SynchronizerMin spacing = 2× dest clock periods
Gray-coded counter2-FF sync (safe)Only 1 bit changes per transition
2-FF Synchronizer
always @(posedge clk_b or posedge rst) if (rst) {sync2, sync1} <= 2'b00; else {sync2, sync1} <= {sync1, signal_from_clk_a}; // sync2 is the safe synchronized output
Rule: Can lose samples → open-loop (2-FF). Every sample matters → closed-loop (handshake/FIFO).

Design Pattern Detector for '1010' — FSM, State Table & Verilog
HardAll▼

States (Overlapping): S0=IDLE, S1=Got'1', S2=Got'10', S3=Got'101', S4=GOT'1010'

StateInput=0Input=1Output
S0S0S10
S1S2S10
S2S0S30
S3S4S10
S4S2S11 (overlap)
Moore FSM — Verilog RTL
module pattern_1010 (input clk, rst_n, din, output reg detect); localparam S0=0, S1=1, S2=2, S3=3, S4=4; reg [2:0] state, nxt; always @(*) begin case (state) S0: nxt = din ? S1 : S0; S1: nxt = din ? S1 : S2; S2: nxt = din ? S3 : S0; S3: nxt = din ? S1 : S4; S4: nxt = din ? S1 : S2; default: nxt = S0; endcase end always @(posedge clk or negedge rst_n) if (!rst_n) state <= S0; else state <= nxt; always @(*) detect = (state == S4); endmodule
Non-Overlapping: At S4, always go to S0: S4: nxt = S0;

Q16
Design a Vending Machine FSM (Candy=4rs, Coke=2rs)
MediumAll▼
module vending_machine ( input clk, rst, coin1, coin2, sel_candy, sel_coke, output reg give_candy, give_coke, give_change ); reg [2:0] balance; always @(posedge clk or posedge rst) begin give_candy <= 0; give_coke <= 0; give_change <= 0; if (rst) balance <= 0; else begin if (coin1) balance <= balance + 1; if (coin2) balance <= balance + 2; end if (sel_coke && balance >= 2) begin give_coke <= 1; balance <= balance - 2; end if (sel_candy && balance >= 4) begin give_candy <= 1; balance <= balance - 4; end end endmodule
Q17
Implement FSM using RAM — concept and Verilog
HardSynopsys▼
RAM Structure: Address = {current_state, input} | Data out = {next_state, output}
module ram_fsm (input clk, rst, din, output reg detect); reg [3:0] mem [0:15]; reg [2:0] state; wire [3:0] addr = {state, din}; wire [3:0] rdata = mem[addr]; always @(posedge clk or posedge rst) if (rst) state <= 3'd0; else state <= rdata[3:1]; assign detect = rdata[0]; endmodule
Advantage: Change behavior by just changing RAM contents — no recompilation needed! Used in configurable protocol engines.
💻 Verilog / SystemVerilog Questions
Q18
Blocking vs Non-Blocking — Deep Dive (Synopsys spent 30 min on this!)
HardSynopsys▼
Blocking (=): Executes sequentially. Use for COMBINATIONAL logic.
Non-Blocking (<=): All RHS evaluated simultaneously. Use for SEQUENTIAL logic.
Swap — Non-Blocking (CORRECT ✅)
always @(posedge clk) begin a <= b; // RHS sampled at same time b <= a; // TRUE SWAP — a=old_b, b=old_a end
Swap — Blocking (WRONG ❌)
always @(posedge clk) begin a = b; // a immediately becomes b b = a; // b becomes new a (which is b!) — BOTH = b end
3-Stage Shift Register — Non-Blocking
always @(posedge clk) begin a <= in; // stage 1 b <= a; // stage 2 — uses OLD a ✅ c <= b; // stage 3 — uses OLD b ✅ end
Golden Rule: Blocking (=) → always @(*). Non-Blocking (<=) → always @(posedge clk). NEVER mix in same block!

Q21
AXI Valid-Ready Handshake Mechanism
MediumQualcomm▼
  • VALID (producer): Asserted when data/address/control is valid and ready to transfer
  • READY (consumer): Asserted when consumer can accept data
  • Transfer happens ONLY when BOTH valid AND ready are HIGH on same clock edge
  • Producer MUST NOT de-assert VALID after asserting it until transfer happens (AXI sticky rule!)
  • READY can be asserted independently of VALID
Timing:
Cycle 1: VALID=1, READY=0 → No transfer (consumer busy)
Cycle 2: VALID=1, READY=1 → ✅ Transfer occurs!
Cycle 3: VALID=0, READY=1 → No transfer (nothing to send)
Q22
Detect One-Hot Encoding from a 32-bit Integer
MediumAll▼
N is one-hot if: N ≠ 0 AND (N & (N−1)) == 0
// N=0b1000 → N-1=0b0111 → N&(N-1)=0 ✅ One-hot! function automatic is_one_hot; input [31:0] data; begin is_one_hot = (data != 0) && ((data & (data - 1)) == 0); end endfunction
Q23
What is a Glitch? How does it propagate and how to fix it?
EasyQualcomm▼

A glitch is a momentary incorrect output in combinational logic caused by different propagation delays.

TypeDescription
Static-1 HazardOutput should stay 1 but momentarily glitches to 0
Static-0 HazardOutput should stay 0 but momentarily glitches to 1
Dynamic HazardOutput changes more than once when it should change only once
  • Add consensus term (redundant cover in K-map)
  • Register outputs — flip-flop filters glitches
  • Use Gray code in counter sequences (only 1 bit changes)
  • Balance paths in physical design (minimize skew)
