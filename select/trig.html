Chapter 9 — Triggers
Overview of Veloce trigger mechanisms for hardware-assisted waveform capture and emulation control.
ℹ️
What are Triggers? Triggers control the hardware trace buffers in the Veloce emulator to capture waveform data around a specific event or time of interest — without unnecessary runtime disruption. Veloce only captures waveforms when trace is ON. When a trigger matures, waveform capture occurs and trace turns OFF automatically.

⚠️
You cannot use both Flexible RTL Triggers and Runtime Triggers at the same time. You also cannot issue hwtrace on/off while an RTL trigger is active.

🎯 Waveform Capture Methods

⏱️ Time-Based Capture
Run the design for a fixed time and capture waveforms during that period. Useful when you know the exact timepoint of a design mismatch.
🔴 Line Breakpoint (LBP)
Set a breakpoint on a specific RTL source line. Emulation stops when the breakpointed line executes. Not supported on StratoCS.
🎯 Trigger-Based Capture
Capture waveforms based on signal values or event sequences. Most flexible — supports store, stop, and conditional triggers.
⚙️
Flexible RTL Triggers
Define triggers at compile time using SystemVerilog RTL subset, then control them at runtime with Tcl commands.

▼
Trigger Module ExampleSYSTEMVERILOG

ConstraintDetail
Ports❌ Trigger modules must have no ports

Incremental RTL Triggers
Add new trigger signals without a full design recompile. Available on Veloce Strato CS only.
ℹ️
Incremental RTL triggers use veltrigcomp to compile triggers independently from the main design database — enabling fast iteration without full recompile.
📝 Step 1 — Signal List Compile
Create and compile the trigger signal list:
Signal ListPLAINTEXT
TRIGPART sync_trig_part GROUP pos_clk_bar_clk CLOCK color_bar_reset_n NET color_bar_vsync NET color_bar_hsync
CompileBASH
comp_veltriggerslist constraints/trig_sig.txt
🔨 Step 2 — Trigger Description Compile
Compile the trigger description file:
veltrigcompBASH
veltrigcomp \ -partition_list trig_part.txt \ -runtime_db_name sync_trig_part_1 # trig_part.txt contents: TRIGPART sync_trig_part TRIGFILES ../src/trig_mod.sv
🚀 Step 3 — Load at Runtime
Load the compiled trigger database:
velrunBASH
velrun ... -veltrigger sync_trig_part_1
🐛 Step 4 — Debug
View trigger signals in Visualizer:
VisualizerTCL
# Compile DB mode vis -veloc -veltrigger <db> # With waveform data vis -tracedir <dir>
⚠️
StratoCS Restriction: Only single-step flow is supported. You cannot use veltrigcomp in StratoCS — a full design recompile is required.

🔄 Resetting a Trigger State Machine
▼
🚫
Only synchronous reset is supported. Trigger compile logic with asynchronous reset is not allowed.
Synchronous Reset ExampleSYSTEMVERILOG
wire rst; // pragma veltrigger reset rst active_high assign rst = top.reset; always @(posedge clk) begin if (rst) begin // Reset assignments for counters and state variables end else begin case (state) // FSM logic endcase end end
Active High: active_high Active Low: active_low
🕐 AllEdgeClk — Unknown Clock Info
▼

When design clock information is unknown, use AllEdgeClk as the clock signal:

Signal List with AllEdgeClkPLAINTEXT
TRIGPART MYGP1 TRIGFILES -f file_list GROUP TMP1 CLOCK POS top_allEdgeClk NET top_counter1 NET top_reset
⚠️
AllEdgeClk can only be used in a posedge-driven sensitivity list. It cannot be used in negedge or combinational assignments.

Veltrigger Tcl Commands & C-APIs
Complete reference for all veltrigger Tcl commands and their C-API equivalents.
ℹ️
C-APIs can be used alongside Tcl commands in the same environment. They work in both clock-running and clock-stopped modes. C-APIs return 0 on success. In multi-comodel mode, C-APIs are only allowed from the primary comodel channel.

📋 Command Reference

veltrigger -download <db> -pos <n> TCL
Download trigger database at position n% in the trace buffer.
veltrigger -enable <name|all> TCL
Enable a specific trigger or all triggers. After download, all triggers are disabled by default.
veltrigger -disable <name|all> TCL
Disable a specific trigger or all triggers.
veltrigger -reset <name|all> TCL
Reset trigger FSM to its initial state. Counters also reset to initial values.
veltrigger -delete TCL
Delete the active RTL trigger. FSM remains active but no longer contributes to stop/trace/trigger.
veltrigger -reload TCL
Reload the trigger database.
veltrigger -display <name> TCL
Show trigger status, counter values, and initial counter values.
veltrigger -getmaturedtrigger TCL
Returns list of triggers that have matured. Cleared after call.
veltrigger -gethaltedtrigger TCL
Returns list of triggers that caused a clock stoppage.
veltrigger -getenabledtrigger TCL
Returns list of currently enabled triggers.
veltrigger -ismatured <name> TCL
Check if a specific trigger has matured.
veltrigger -ishalted <name> TCL
Check if a specific trigger is in halted state.
veltrigger -isenabled <name> TCL
Check if a specific trigger is enabled.
veltrigger -getval <name.counter> TCL
Get the current value of a counter in a trigger module.
veltrigger -setval <name.counter> <val> TCL
Set the value of a counter in a trigger module.
veltrigger -setinitial <name.counter> <val> TCL
Set the initial value for a counter (used on reset).
veltrigger -cleartriggerstate TCL
Clear current trigger state to allow processing of new $trigger calls.
veltrigger -onhalt <proc_name> TCL
Register a Tcl callback procedure that executes when a trigger stops the clock. If return value is 0, clock continues automatically.

⏱️ Timer Declaration
▼
Timer SyntaxPLAINTEXT
timer TooLong begin value = 10000; // Count at vclock rate target = TooLong; // State to jump to when fired end

Timer Actions on Transitions:

enable timer <name> disable timer <name> reset timer <name>
💡
Timers count at vclock rate when enabled. When the counter matches the specified value, the FSM immediately enters the timer's target state.
🔢 Counter Declaration
▼
Counter SyntaxPLAINTEXT
// Simple counter — no properties needed counter Loopy; // With pragma (for display and control): reg [31:0] counter1; // pragma veltrigger counter counter1

Counter Actions on Transitions:

increment counter <name> reset counter <name> condition (<name> == <value>)
📦 State Declaration & Attributes
▼
State SyntaxPLAINTEXT
state <name> : <attribute_list> begin domain <domain_name>; next (<condition>) [N times] [counter/timer actions]; jump (<condition>) [N times] [target = <state>] [actions]; end

State Attribute Reference:

AttributeDescription
initialMarks the starting state. Only one allowed. Default = first state.
triggerMarks a trigger maturity state. Multiple allowed.
storeTurns waveform storage ON
nostoreTurns waveform storage OFF
store if <expr>Enables storage conditionally based on expression
stopStops emulation clocks when state is reached
stop if <expr>Stops emulation clocks conditionally
⚠️
Limits: Max 4 unique constants per counter (including N times). Max 4 transitions (next + jump combined) per state. More than 4 transitions results in an error.
🌐 Domain Declaration
▼
Domain SyntaxPLAINTEXT
// Global domain (applies to all states without their own domain) domain <name> @(posedge <clk>); domain <name> @(negedge <clk>); // Per-state domain (inside state block) state MyState begin domain myDomain @(posedge clk); next (condition); end
💡
For VHDL or mixed VHDL/Verilog designs, use the slash (/) as the hierarchy separator for signal names in trigger descriptions.

Trigger Position
Control where in the trace buffer the trigger event falls — from 0% (start) to 100% (end).
🎛️ Interactive Trigger Position Simulator — drag the slider to adjust:
Pre-Trigger (50%)
Post-Trigger (50%)
50%
Trigger occurs midway — half of trace before, half after trigger matures.
📌 Position = 0%
Trigger occurs at the start of trace collection. The signal window includes only a few samples before the trigger.
📌 Position = 50% (Default)
Trigger occurs midway in trace collection. Half of the waveform before the trigger, half after.
📌 Position = 100%
Trigger occurs at the end of trace collection. The signal window includes only a few samples after the trigger.
Setting Position via TclTCL
veltrigger -download <db_name> -pos 75
ℹ️
Sample transcript message:
Info! [RTS-20065] : Trigger position: 50, Trace will turn off after 23904 cycles of master clock after trigger matures. Total trace depth: 47808 cycles of master clock.
⚠️
Edge Case: If the trigger matures before the trace buffer reaches the set position %, only the available data is captured. The trace buffer may not fill to 100%.

Trigger Examples
13 annotated examples covering all major trigger patterns. Click a tab to view each example.
📌
Store Trigger: Captures waveforms around the trigger without stopping emulation. Trace continues running.
Example 1 — Store TriggerPLAINTEXT
domain domain0 @(posedge Uclock); state S0 : initial, store begin next (top.SIM_TIME[63:0] == 100); end state S1 : trigger, store;
📌
Stop Trigger: Stops emulation as soon as the trigger matures. Clocks halt immediately.
Example 2 — Stop TriggerPLAINTEXT
state S0 : initial, store begin next (top.SIM_TIME[63:0] == 100); end state S1 : trigger, stop;
📌
Stop If: Emulation stops when the signal takes value 8'h55 before it takes 8'h65 (after having taken 8'h40).
Example 3 — Conditional StopPLAINTEXT
state StartState : initial next (counter_test.sr.data[7:0] == 8'h40); state ConditionStop : stop if (counter_test.sr.data[7:0] == 8'h55), store begin domain Clk_dom1 @(posedge clk); next (counter_test.sr.data[7:0] == 8'h65); end state NormalTrigger : trigger;
📌
N Contiguous Times: The jump resets the counter if the condition ever becomes false — ensuring truly contiguous matching.
Example 4 — Match N Contiguous TimesPLAINTEXT
state AState begin next (one.two.address[31:0] == 32'h0A0A0A0A) 49 times; jump (one.two.address[31:0] != 32'h0A0A0A0A) target = AState; end
📌
Don't Care: Use X or x in bit patterns to create don't-care conditions. Only specified bits are checked.
Example 5 — Don't Care ConditionalPLAINTEXT
state First_Dont_Care_Check : initial next (shift_test_rising.sr0.dataOut[3:0] == 4'bX0x1); state Second_Dont_Care_Check next (shift_test_rising.sr0.dataOut[3:0] == 4'b1x0x); state Final : trigger;
📌
Wait N Clock Ticks: Use next (1) N times to wait for a fixed number of clock cycles.
Example 6 — Wait N Clock TicksPLAINTEXT
// Simple single-domain version: state WaitAround next (1) 100 times; // Multi-domain explicit version: state WaitAround begin domain Simple @(posedge CLK); next (1) 100 times; end
📌
Timer Timeout: Use a timer and a unique timer trigger state to detect when a normal trigger does not occur within an expected period.
Example 7 — Timer-Based TimeoutPLAINTEXT
timer TooLong begin value = 10000; target = TooLong; end state Begin next (SomethingOrOther == 16'hABCD) enable timer TooLong; state NormalTrigger : trigger; state TooLong : trigger;
📌
Counter Loop: Use a general-purpose counter to repeat a pattern exactly N times.
Example 8 — Repeat Pattern N TimesPLAINTEXT
counter Loopy; state Start begin next (1) reset counter Loopy; end state StartOfLoop next (SomethingOrOther == 32'h1234) increment counter Loopy; state EndOfLoop begin next (1) condition (Loopy == 17); jump (1) target = StartOfLoop; end
📌
Custom Initial State: Use the initial attribute to specify a starting state that is not the first state in the file.
Example 9 — Specify Initial StatePLAINTEXT
state One next (WhatHaveYou == 0); state Two next (SomethingElse == 1); // ... state ActuallyStartHere : initial // ...
📌
Absolute Jump: Trigger states of the jump instruction are specified by their exact state names.
Example 10 — Absolute Jump TriggerPLAINTEXT
state S1 : initial begin jump (sr1.dataOut[2:0] == 3'b111) target = S1; jump (sr1.dataOut[2:0] == 3'b101) target = S3; end state S2 begin jump (sr1.dataOut[2:0] == 3'b000) target = S1; jump (sr1.dataOut[2:0] == 3'b011) target = final; end state S3 jump (sr1.dataOut[2:0] == 3'b100) target = S4; state S4 jump (sr1.dataOut[2:0] == 3'b010) target = S2; state final : trigger;
📌
Relative Jump: Target state is specified by its position relative to the current state. 0 = self, -1 = previous, +2 = two states forward.
Example 11 — Relative Jump TriggerPLAINTEXT
state S1 : initial begin jump (sr1.dataOut[2:0] == 3'b111) target = 0; // self jump (sr1.dataOut[2:0] == 3'b101) target = 2; // +2 states forward end state S2 begin jump (sr1.dataOut[2:0] == 3'b000) target = -1; // previous state jump (sr1.dataOut[2:0] == 3'b011) target = 3; // +3 states forward end state final : trigger;
📌
Expression Types: Triggers support logical operators, array expressions, and reduction operators.
Example 12a — Logic OperatorsPLAINTEXT
state S0 : initial next ((sr0.dataOut[3:0] == 4'b1010) && (sr1.dataOut[3:0] == 4'b1100) || sr1.dataOut[3:0] == 4'b0101) 2 times;
Example 12b — Array ExpressionsPLAINTEXT
state S0 : initial next (sr0.dataOut[10-3 : 3+1] == 4'b1010); state S1 next (sr0.dataOut[(2*5+3)/6-1] == 1'b1);
Example 12c — Reduction OperatorsPLAINTEXT
state Start : initial next (sr0.dataOut[3:0] == 4'hF); state S0 next (&sr0.dataOut[3:0]) 2 times; // AND reduction
📌
Full Feature Tour: Annotated example showing domains, timers, counters, store if, nostore, conditions, and multiple trigger states.
Example 13 — Various Triggering FeaturesPLAINTEXT
// Trigger at 30% point of probe window domain single @(posedge clk); timer One begin value = 100; target = TimerOneExpired; end counter Meaningless; state Start : initial next (c[0:2] == 3) reset timer One; state One next (d[7:8] == 2); state Two : nostore begin domain yo; next (some == 8'b0) 3 times; jump (c[0:2] == 0) 2 times begin target = One; increment counter Meaningless; end end state Three : store if (enable == 1'b0) begin next (d == 4'h7 && c[1] == 1'b0) begin condition Meaningless == 23; disable timer One; end end state Final : trigger; state TimerOneExpired : trigger;

📝 Basic Setup
▼
Memory Fill on EmptyTCL
# Add event — fires when top.mem0.empty == 1 set event0 [hwevent add top.mem0.empty 1] hwevent onmature -proc fillmem proc fillmem {} { global event0 if {[hwevent ismatured $event0]} { memory download -instance top.mem0.mem -file abc.txt \ -addr_range 0:65535 -apply reg setvalue top.mem0.empty 0 -apply } }
🔄 Auto Trace On/Off with HW Events
▼
Auto Trace ToggleTCL
set event0 [hwevent add top.a.event0 1] hwevent onmature -proc autoOnTest proc autoOnTest {} { global event0 event1 if {[hwevent ismatured $event0]} { hwetrace autoupload on -tracedir my_dir hwevent delete $event0 set event1 [hwevent add top.a.event0 0] hwevent onmature -proc autoOffTest } } proc autoOffTest {} { global event1 event0 if {[hwevent ismatured $event1]} { hwetrace autoupload off hwevent delete $event1 set event0 [hwevent add top.a.event0 1] hwevent onmature -proc autoOnTest } }
💀 Fatal Event Stop (C-API)
▼
Fatal Event C-APIC
int eventfatal = dm_event_add("top.a.fatal", 1); int event0 = dm_event_add("top.a.event0", 1); dm_onmature_callback(testMain); int testMain() { if (dm_event_ismatured(eventfatal)) { dm_checkpoint_save("logcheck", true, false); return 1; // Non-zero = STOP emulation } if (dm_event_ismatured(event0)) { dm_memory_download(...); } return 0; // Zero = CONTINUE emulation }
⚠️ HW Event Restrictions
▼
RestrictionDetail
Dynamic ICE models❌ Events cannot support dynamic ICE models (events stop clocks)

💡 Common HW Event Use Cases

💾 Memory Fill on Empty
Detect FIFO empty condition → automatically download new data into memory. Keeps the emulation running without manual intervention.
📤 Memory Upload on Full
Detect FIFO full condition → automatically upload memory contents to the host for analysis.
🔧 Register Force/Release
Force or release register values based on signal conditions using reg setvalue commands inside the callback.
🌊 Auto Waveform Upload
Automatically upload waveforms when a specific event fires, enabling continuous waveform capture without manual steps.
🔴 Fatal Error Stop
Stop emulation immediately on a fatal signal condition. Return non-zero from the callback to halt the emulation run.
📊 Conditional Logic Control
Run complex Tcl or C logic based on multiple simultaneous events using ismatured checks inside the callback.
